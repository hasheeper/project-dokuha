<variable_rule>
【DOKUHA 变量说明】

变量根路径：`stat_data`

可写 JSONPatch 路径：
```text
/dokuha/familiarity/points
/dokuha/coreStates/relationshipStage
/dokuha/coreStates/attachmentLevel
/dokuha/mentalStates/disorderActive
/dokuha/mentalStates/longTermEmotion
/dokuha/mentalStates/dynamicEmotion
/dokuha/outfit
/dokuha/accessories
/dokuha/current_location
/dokuha/current_event/phase
/dokuha/context_notes/pending_event_summary
/system/time_advance
/system/time_set_to
/system/event_start/name
/system/event_start/type
```

字段说明：
- `/dokuha/familiarity/points`: 熟悉度点数，范围 `0..500`；固定使用 `delta`，`value` 写本轮增减量。
- `/dokuha/coreStates/relationshipStage`: 当前关系阶段。
  - `neighbor`: 邻居。
  - `friend`: 朋友。
  - `lover`: 恋人。
- `/dokuha/coreStates/attachmentLevel`: 当前依恋阶段。
  - `non_attached`: 非依恋。
  - `light_attached`: 轻依恋。
  - `heavy_attached`: 重依恋。
- `/dokuha/mentalStates/disorderActive`: 当前病理触发数组；无触发时为 `[]`。
  - `asd_active`: ASD 触发。
  - `adhd_active`: ADHD 触发。
  - `bpd_active`: BPD 触发。
  - `pmdd_active`: PMDD 触发。
- `/dokuha/mentalStates/longTermEmotion`: 长期情绪底色。
  - `depressed`: 低落。
  - `exhausted`: 耗竭。
  - `normal`: 平常。
  - `comfortable`: 舒适。
  - `irritated`: 烦躁。
  - `paralyzed`: 麻木。
- `/dokuha/mentalStates/dynamicEmotion`: 当前互动后的动态态度。
  - `normal`: 平静。
  - `warm`: 柔和。
  - `passionate`: 热切。
  - `slightly_cold`: 微冷。
  - `freezing_cold`: 冰冷。
- `/dokuha/outfit`: 当前基础服装底座。
  - `streetwear_full`: 完整常服。
  - `streetwear_inner`: 常服内层。
  - `nightwear`: 睡衣。
  - `underwear`: 内衣。
  - `nude`: 裸体。
- `/dokuha/accessories`: 当前基础配件数组。
  - mask 状态三选一：`no_mask | mask_up | mask_down`
  - 可额外叠加：`headphones`
- `/dokuha/current_location`: 当前地点，使用 PascalCase 英文地点名，例如 `ApartmentHallway`、`DokuhaRoom`。
- `/dokuha/current_event/phase`: 当前事件结束时写为 `end`。
- `/dokuha/context_notes/pending_event_summary`: 事件结束摘要；与 `/dokuha/current_event/phase = end` 同轮写入，写清事件结果、关系影响或后续记忆点。
- `/system/time_advance`: 时间推进命令，字符串格式为 `30 min`、`2 hr`、`1 day`、`1 week`、`1 month`。
- `/system/time_set_to`: 时间定位命令，字符串格式为 `23:00`、`D+1 09:00`、`Wed 21:15`、`2024-4-1 20:00`。
- `/system/event_start/name`: 新事件开启命令的事件名，使用 PascalCase 英文。
- `/system/event_start/type`: 新事件开启命令的事件类型。
  - `daily_event`: 日常事件。
  - `relationship_event`: 关系事件。
  - `dokuha_crisis_event`: 毒羽危机事件。
  - `pmdd_event`: PMDD 相关事件。
  - `bad_luck`: 倒霉事件。

事件启动说明：
- 当 `/dokuha/current_event/type = none` 且本轮正文已经形成具体情境、请求、麻烦、冲突、照顾、邀约或危机时，使用 `/system/event_start/name` 与 `/system/event_start/type` 启动事件。
- 事件启动写入 `/system/event_start/name` 与 `/system/event_start/type`。
- 普通寒暄、短暂吐槽或没有可命名目标的自由对话，不启动事件。

JSONPatch 格式：
- 数值增减使用 `delta`。
- 字段替换使用 `replace`。
- 变量更新块固定使用 `<UpdateVariable><Analysis>...</Analysis><JSONPatch>...</JSONPatch></UpdateVariable>`。
- 无变量更新时输出 `Skipping`。

示例：
```xml
<UpdateVariable>
<Analysis>本轮有明确照顾或关系推进，熟悉度长期上升。</Analysis>
<JSONPatch>[{"op":"delta","path":"/dokuha/familiarity/points","value":15}]</JSONPatch>
</UpdateVariable>
```

```xml
<UpdateVariable>
<Analysis>本轮互动后，毒羽的即时态度稳定转为柔和。</Analysis>
<JSONPatch>[{"op":"replace","path":"/dokuha/mentalStates/dynamicEmotion","value":"warm"}]</JSONPatch>
</UpdateVariable>
```

```xml
<UpdateVariable>
<Analysis>正文中口罩被拉下，同时耳机仍然佩戴。</Analysis>
<JSONPatch>[{"op":"replace","path":"/dokuha/accessories","value":["mask_down","headphones"]}]</JSONPatch>
</UpdateVariable>
```

```xml
<UpdateVariable>
<Analysis>场景实际移动到公寓门口，并推进了约十五分钟。</Analysis>
<JSONPatch>[{"op":"replace","path":"/system/time_advance","value":"15 min"},{"op":"replace","path":"/dokuha/current_location","value":"ApartmentHallway"}]</JSONPatch>
</UpdateVariable>
```

```xml
<UpdateVariable>
<Analysis>正文触发了新的日常事件开启契机。</Analysis>
<JSONPatch>[{"op":"replace","path":"/system/event_start/name","value":"HallwayEncounter"},{"op":"replace","path":"/system/event_start/type","value":"daily_event"}]</JSONPatch>
</UpdateVariable>
```

</variable_rule>
